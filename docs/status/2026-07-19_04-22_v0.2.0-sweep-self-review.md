## a) FULLY DONE (verified green: fmt + clippy default+encryption + 40 unit + 15 doc tests + doc build + examples run)

1. **Envelope false-positive hardening** (`src/segment.rs`) — `unwrap_envelope` now requires the `SBF1` magic AND the 3 reserved bytes to all be zero. Drops the false-positive rate on legacy encrypted files from 2⁻³² (~1 per 7 monitor365 deployments) to 2⁻⁵⁶ (negligible across the full 597M-segment corpus). Regression test `envelope_detection_requires_zero_reserved_bytes` proves the hardening fires. Module doc updated to match.
2. **Encrypted-legacy read coverage** (`src/tests.rs`) — added `legacy_encrypted_file_without_envelope_still_reads`, which hand-crafts a `[nonce][ciphertext]` file with no envelope and proves it reads back through the enveloped reader with the matching cipher. The headline monitor365 byte-compatibility guarantee now has actual test coverage (was zero before).
3. **Proptest key variation** (`src/property_tests.rs`) — `full_write_read_encrypted_roundtrip` now takes `key in any::<[u8; 32]>()` per case instead of the fixed `[0x42; 32]`. Catches key-dependent AEAD edge cases.
4. **`recover()` refactor** (`src/lib.rs`) — all `fs::metadata` I/O now happens BEFORE the mutex is taken; the lock is held only to publish `head_seq`/`next_seq`/`approx_disk_bytes`. Restores the "mutex never held across I/O" invariant the prior review flagged as violated in spirit.
